树与二叉树的应用

哈夫曼树与哈夫曼编码

概念

哈夫曼树 （Huffman Tree）是带权路径长度（WPL）最小的二叉树 ，又称为最优二叉树。

  • 带权路径长度 ：树中所有叶子节点的权值与路径长度的乘积之和。

其中

  • ：第 个叶子节点的权值。

  • ​：根节点到第 个叶子节点的路径长度。

构建哈夫曼树的步骤

  1. 将每个权值作为一个独立的节点，构成森林。

  2. 选取权值最小的两个节点，合并为一个新的父节点，父节点的权值为子节点的权值之和。

  3. 重复步骤2，直到只剩下一个根节点，形成哈夫曼树。

哈夫曼编码

定义 ：基于哈夫曼树生成的前缀编码 （任一字符的编码都不是其他字符编码的前缀）。

应用

  • 数据压缩：如文件压缩（WinRAR）。

  • 信息传输：减少传输的位数。

二叉查找树（BST）

概念与特点

二叉查找树是一种特殊的二叉树，具有以下性质：

  1. 若左子树非空，则左子树上所有节点的值均小于根节点的值。

  2. 若右子树非空，则右子树上所有节点的值均大于根节点的值。

  3. 左、右子树也分别是二叉查找树。

实现

插入操作

  • 从根节点开始，依次比较节点值与插入值。

  • 小于当前节点值，进入左子树；大于当前节点值，进入右子树。

  • 直到找到空位置，插入新节点。 代码示例

TreeNode* Insert(TreeNode* root, int value) { if (root == NULL) { root = (TreeNode*)malloc(sizeof(TreeNode)); root->data = value; root->left = root->right = NULL; } else if (value < root->data) { root->left = Insert(root->left, value); } else if (value > root->data) { root->right = Insert(root->right, value); } return root; }

并查集

概念与特点

并查集（Union-Find）是一种树型的数据结构，用于处理动态连通性问题。

  • 主要功能：

    1. 合并（Union） ：将两个元素所属的集合合并。

    2. 查找（Find） ：找到元素所属集合的代表元 （根节点）。

实现

并查集通常使用数组或树实现。每个集合用一棵树表示，根节点为集合的代表元素。

核心操作

  1. 查找（Find）

    • 查找元素所在集合的根节点。

  2. 合并（Union）

    • 连接两个集合的根节点，将其中一个集合挂到另一个集合上。

压缩查找路径的方法

路径压缩优化 Find 操作，减少树的高度，提高效率。

  • 方法 ：在查找过程中，将访问路径上的节点直接指向根节点。

int Find(int parent[], int x) { if (parent[x] != x) { parent[x] = Find(parent, parent[x]); // 路径压缩 } return parent[x]; }

时间复杂度 ：接近 O(1)（采用路径压缩和按秩合并）。

堆

堆的概念和特性

堆（Heap）是一棵完全二叉树 ，满足以下性质：

  1. 大根堆 ：根节点的值 ≥ 子节点的值。

  2. 小根堆 ：根节点的值 ≤ 子节点的值。

堆的特点：

  • 每个子树也是堆。

  • 适合用于优先队列 、排序等。

最大堆与最小堆

  • 最大堆 ：根节点存储最大值。

  • 最小堆 ：根节点存储最小值。

堆的实现

  1. 顺序存储 ：使用数组存储堆，根节点在数组下标1。

    • 左子节点：2i2i2i

    • 右子节点：2i+12i + 12i+1

    • 父节点：i/2i / 2i/2

  2. 插入操作：

    • 将新元素插入堆的末尾，然后上浮调整堆。

  3. 删除操作 （删除根节点）：

    • 将堆尾元素替换根节点，然后下沉调整堆。

二叉搜索树（BST）

概念与特点 ：见前述内容。

平衡树

概念 ：平衡树是一种保证树的高度接近最小的二叉搜索树，常见的有AVL树、 红黑树等。

  • AVL树 ：每个节点的左右子树高度差的绝对值不超过1。

  • 红黑树 ：具有平衡特性的二叉查找树，颜色标记节点（红/黑）。

B树

概念

B树是一种多路搜索树 ，每个节点可以有多个子节点。适合用于磁盘存储和大规模数据检索。

特点

  1. 树的高度较低 ，减少磁盘 I/O 操作。

  2. 每个节点可以存储多个关键字，子节点数为 mmm（B树的阶）。

  3. 关键字有序：

    • 左子树关键字 < 当前关键字 < 右子树关键字。

  4. 所有叶子节点在同一层。

B+树（了解概念即可）

概念

B+树是B树的变种，具有以下特点：

  1. 非叶子节点仅存储索引 ，所有关键字和数据存储在叶子节点。

  2. 叶子节点通过链表连接，支持区间查找。

应用 ：数据库索引、文件系统等。

2-3-4树/红黑树(不做考查)

概念

2-3-4树是2-3树的扩展，具有以下特点：

  1. 每个节点可以存储1、2 或 3个关键字 ，有2、3或4个子节点。

  2. 树是平衡的，所有叶子节点在同一层。

  3. 满足多路搜索树的有序性。

关系 ：2-3-4树与红黑树等价。

总结

  1. 并查集 ：解决动态连通性问题，优化方法有路径压缩和按秩合并。

  2. 堆 ：用于优先队列与排序，分为最大堆和最小堆。

  3. 二叉搜索树（BST） ：动态查找效率高，但可能退化为线性结构。

  4. 平衡树 ：如AVL树、红黑树，解决BST不平衡问题。

  5. B树/B+树 ：适合磁盘存储与大规模数据索引。

  6. 2-3树/2-3-4树 ：平衡的多路搜索树，红黑树与2-3-4树等价。
