图

概念与相关概念

定义

图（Graph）是由顶点和边组成的数据结构，表示元素之间的关系。 表示：

其中：

  • ：顶点集合

  • ：边的集合

分类

  1. 有向图 ：边有方向，表示从一个顶点到另一个顶点的单向关系。

  2. 无向图 ：边无方向，表示两个顶点之间的双向关系。

  3. 带权图 ：边带有权值，表示关系的权重。

相关概念

  • 度：

    • 顶点的度：与该顶点相关联的边的数目。

    • 入度/出度：有向图中顶点的入边数/出边数。

  • 路径 ：顶点序列，依次通过边连接。

  • 简单路径 ：路径中不包含重复顶点。

  • 回路/环 ：起点与终点相同的路径。

  • 连通图 ：任意两个顶点之间都有路径相连的图。

存储

邻接矩阵

  • 使用二维数组存储图：

    • 无向图 ：对称矩阵。

    • 有向图 ：非对称矩阵。

    • 带权图 ：数组元素表示权值。

邻接表

  • 每个顶点对应一个链表 ，链表存储该顶点的所有邻接顶点。

示例：

优缺点：

  • 优点 ：节省空间，适合稀疏图。

  • 缺点 ：查找边不如邻接矩阵高效。

邻接多重表

  • 适用于无向图，去除边的重复存储，存储边信息更加高效。

  • 每条边用一个节点表示，包含两个顶点的指针。

图的遍历

BFS（广度优先搜索）

概念 ：类似于层次遍历，按层访问顶点，使用队列实现。

过程：

  1. 从起始顶点开始，访问并标记已访问。

  2. 将与该顶点相邻且未访问的顶点加入队列。

  3. 依次出队，重复步骤2，直到队列为空。

时间复杂度：

DFS（深度优先搜索）

概念 ：类似于树的先序遍历，使用栈或递归实现。

过程：

  1. 从起始顶点开始，访问并标记已访问。

  2. 访问下一个未访问的邻接顶点，继续递归。

  3. 若当前顶点无未访问的邻接顶点，回溯到上一个顶点。

图的连通性

  1. 连通图 ：无向图中，任意两个顶点之间都有路径。

  2. 强连通图 ：有向图中，任意两个顶点之间都有双向路径。

最小生成树问题

概念

最小生成树（MST）：连通图中使所有顶点连通的权值之和最小的树。

算法

  1. Prim算法 ：适合稠密图。

    • 逐步扩展顶点集合，选择权值最小的边。

    • 时间复杂度： （使用邻接矩阵）。

  2. Kruskal算法 ：适合稀疏图。

    • 按边的权值排序，逐步选取不会形成回路的边。

    • 时间复杂度： 。

最短路径问题

迪杰斯特拉算法（Dijkstra）

目标 ：求单源最短路径。
特点：

  • 适用于带权有向图 ，要求权值非负。

  • 使用贪心思想 ：每次选择当前距离起点最近的顶点。

时间复杂度： ，使用优先队列可优化为 。

Floyd算法

目标 ：求所有顶点之间的最短路径（多源最短路径）。
思想 ：动态规划，逐步更新顶点间最短路径。

时间复杂度： 。

拓扑排序问题

概念

  • 拓扑排序是对有向无环图 （DAG）的顶点进行排序，使得每条边 (u, v) 中，顶点 uuu 排在顶点 vvv 之前。

算法

  1. 计算每个顶点的入度。

  2. 将入度为0的顶点加入队列。

  3. 出队顶点，并将其邻接顶点的入度减1。

  4. 重复步骤2，直到队列为空。

时间复杂度： 。

关键路径问题

概念

关键路径是带权有向无环图（DAG） 中从起点到终点的最长路径 ，用于确定工程项目的最早完成时间。

算法实现

  1. 求顶点的最早开始时间 （正向拓扑排序）。

  2. 求顶点的最晚开始时间 （反向拓扑排序）。

  3. 找到所有关键活动：

    • 最早开始时间 = 最晚开始时间 的活动为关键路径上的活动。

总结

  1. 图的存储 ：邻接矩阵适合稠密图，邻接表适合稀疏图。

  2. 遍历 ：BFS 和 DFS 是图的基础算法，理解其过程与实现。

  3. 最短路径 ：Dijkstra 解决单源最短路径，Floyd 解决多源最短路径。

  4. 最小生成树 ：Prim 和 Kruskal 分别适用于稠密图和稀疏图。

  5. 拓扑排序 和 关键路径 ：用于有向无环图（DAG）。
