栈和队列

栈

概念和特性

  1. 定义 ：栈（Stack）是一种先进后出 （LIFO, Last In First Out）的线性数据结构。

  2. 特性：

    • 插入 和 删除 操作仅在栈顶进行。

    • 栈的基本操作包括：入栈（Push）、出栈（Pop）、取栈顶元素（Top/Peek）。

  3. 应用：

    • 括号匹配、 表达式求值、 递归调用 等。

各种操作

操作

描述

Push

将元素压入栈顶。

Pop

将栈顶元素弹出。

Top

返回栈顶元素，但不删除该元素。

IsEmpty

判断栈是否为空。

顺序表栈

  1. 实现 ：使用数组实现栈，栈顶指针标识当前栈顶的位置。

  2. 优缺点：

    • 优点 ：实现简单，直接用数组即可。

    • 缺点 ：栈的大小固定，空间不灵活。

  3. 时间复杂度：

    • Push、Pop、Top 操作均为 O(1)。

链表栈

  1. 实现 ：使用单链表实现栈。

    • 栈顶对应链表的头节点。

  2. 优缺点：

    • 优点 ：无需预分配空间，空间利用率高。

    • 缺点 ：需要额外存储指针域，略占空间。

  3. 时间复杂度：

    • Push、Pop、Top 操作均为 O(1)。

双栈共享空间

概念 ：在一块连续的存储空间中，使用两个栈：

  • 一个栈从左向右增长，另一个栈从右向左增长。

  • 目的 ：节省空间，提高存储效率。

实现关键

  • 维护两个栈顶指针： top1 和 top2。

  • 栈满条件： top1 + 1 == top2。

出栈入栈问题与其他问题

  1. 栈序列判断 ：给定一个入栈序列，判断一个出栈序列是否合法。

  2. 栈的应用问题 ：如括号匹配、表达式求值等。

队列（循环队列）

概念和特性

  1. 定义 ：队列（Queue）是一种先进先出 （FIFO, First In First Out）的线性数据结构。

  2. 特性：

    • 数据从队尾插入（Enqueue）。

    • 数据从队头删除（Dequeue）。

  3. 分类：

    • 普通队列 ：队尾插入，队头删除。

    • 循环队列 ：解决普通队列“假溢出”的问题。

各种操作

操作

描述

Enqueue

向队尾插入元素。

Dequeue

从队头删除元素。

IsEmpty

判断队列是否为空。

IsFull

判断队列是否满（循环队列）。

顺序表队列

  1. 实现 ：使用数组实现队列。

  2. 问题 ：当元素出队后，队头前面的空间无法再利用（假溢出）。

  3. 解决方案 ：采用循环队列。

循环队列

  1. 实现：

    • 维护两个指针： front （队头） 和 rear （队尾）。

    • 队空条件 ：front == rear。

    • 队满条件： (rear + 1) % capacity == front。

  2. 优势 ：解决了顺序队列的“假溢出”问题。

栈和队列的应用

括号匹配

问题 ：给定一个表达式，检查括号是否匹配。
解法：

  • 使用栈，遇到左括号入栈，遇到右括号检查栈顶元素是否匹配。

  • 最后判断栈是否为空。

表达式转换与求值

  1. 中缀表达式转后缀表达式 （逆波兰表达式）：

    • 使用栈保存操作符，保证优先级。

  2. 后缀表达式求值：

    • 遇到数字入栈，遇到操作符弹出两个数字进行计算，结果入栈。

八皇后问题

问题 ：在 8×8 的国际象棋棋盘上放置 8 个皇后，使得任何两个皇后都不在同一行、列或对角线上。
解法：

  • 使用栈或递归进行回溯搜索。

递归程序

递归的设计

递归的三要素：

  1. 边界条件 ：结束递归的条件。

  2. 递归公式 ：定义当前问题与子问题的关系。

  3. 递归过程 ：逐步逼近边界条件。

尾递归优化

优先队列

概念

优先队列是一种特殊的队列，每个元素都有一个优先级 ，优先级高的元素先出队。

实现

  1. 顺序存储 ：用数组或链表实现。

    • 插入：O(1)

    • 删除（取出最大/最小优先级）：O(n)

  2. 堆 （推荐）：

    • 小顶堆/大顶堆 实现。

    • 插入和删除操作的时间复杂度均为 O(log n)。
